# 博客 403 排查：Nginx 路径链 + 构建产物异常

date: 2026-07-15
tags: [博客, 运维, 403, Nginx, 构建, 排查]

---

## 现象

构建成功后访问 `https://blog.zzr233.online` 返回 **403 Forbidden**。

## 排查过程

### 1. 容器名错误（历史问题，这次没犯）

之前犯过：容器名是 `nginx-home` 不是 `nginx`，`docker restart nginx` 不生效。  
这次确认正确 ✅

### 2. 完整路径链梳理

```
源码目录                          Docker bind mount           Nginx 容器内
/home/ubuntu/blog/               → 挂载到容器               → 提供服务
  ├── astro build 输出到 dist/   → /var/www/blog/html       → root /var/www/blog/html
  ├── dist/                                              
  │   ├── index.html                                     → try_files $uri $uri/ /index.html
  │   ├── _astro/                                        → 静态资源
  │   └── posts/                                         → 文章页面
  └── docker inspect 确认挂载正确 ✅
```

**关键挂载**：`docker inspect nginx-home` → `/home/ubuntu/blog/dist` → `/var/www/blog/html`

**关键 Nginx 配置**：
```nginx
root /var/www/blog/html;
index index.html;
location / {
    try_files $uri $uri/ /index.html =404;
}
```

### 3. 根因：dist/ 缺少 index.html

首次检查 dist/ 内容：
```
drwxrwxr-x  _astro/
drwxrwxr-x  .prerender/     ← 异常！SSR 模式才有这个目录
```

- ❌ 没有 `index.html`
- ❌ 没有 `posts/`、`about/`、`tags/` 等页面目录
- ❌ 多了一个 `.prerender/` 目录（Astro hybrid/SSR 产物）
- Nginx 访问 `/` → 找不到 `index.html` → **403**

重建后正常 dist/ 内容：
```
index.html         (407KB, 正常工作)
_astro/
posts/
about/
tags/
archive/
... (共 23 个页面)
```

### 4. `.prerender/` 出现的原因（推测）

astro.config.mjs 配置为 `output: "static"`，不应出现 `.prerender/`。

**推测**：之前的 `cat >>` 命令误操作或文件编辑导致某个文件损坏，Astro 在构建时 fallback 到 hybrid 模式，产出了 `.prerender/` 替代正常静态页面。

**具体操作追溯**：
- 编辑 `PostPage.astro` 时尝试用 `cat >> src/layouts/PostPage.astro`（错误路径）
- 实际文件在 `src/components/layout/PostPage.astro`
- 虽然错误路径没创建成功，但正确路径的文件可能在编辑过程中损坏

### 5. 重建恢复正常

重新执行 `pnpm run build`（完整构建链）：
```
generate-icons.js → generate-lqips.ts → astro build → subset-fonts.ts → pagefind
```

构建输出：`23 page(s) built in 1m 41s` ✅  
`index.html` 正常生成，Pagefind 索引 12 页、1443 词。

## 教训

### 路径类（反复犯，要记住）

| 问题 | 正确 |
|------|------|
| 容器名 `nginx` | ❌ → `nginx-home` ✅ |
| mount 路径 | ✅ `/home/ubuntu/blog/dist` → `/var/www/blog/html` |
| Nginx root | ✅ `/var/www/blog/html` |

### 构建产物异常排查 checklist

如果部署后 403，按顺序查：

1. **`docker ps`** 确认容器名和运行状态
2. **`docker inspect <容器名>`** 确认挂载路径
3. **`ls dist/`** 确认有 `index.html`（不是 `.prerender/`）
4. **`docker exec <容器名> cat /etc/nginx/conf.d/blog.conf`** 确认 root 路径
5. **构建日志**：检查是否有非致命错误（如 Google Fonts 超时只是 warning，不影响构建）

### 预防

- 编辑文件前先 `find` 确认正确路径
- 不要在 `dist/` 存在时中断构建（会留下不完整产物）
- 用 `patch` 工具代替 `cat >>` 编辑文件，避免路径写错

## 相关命令备忘

```bash
# 查容器挂载
docker inspect nginx-home --format '{{range .Mounts}}{{.Source}} -> {{.Destination}}{{"\n"}}{{end}}'

# 查 Nginx 配置
docker exec nginx-home cat /etc/nginx/conf.d/blog.conf

# 完整构建
cd /home/ubuntu/blog && pnpm run build

# 重启容器（让 Nginx 重新加载 dist/）
docker restart nginx-home
```
