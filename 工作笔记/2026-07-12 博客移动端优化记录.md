## 1. `www.zzr233.online` 证书错误 ❌ → ✅

**场景：** 用户手机访问 `zzr233.online` 提示「不安全」（`NET::ERR_CERT_COMMON_NAME_INVALID`），具体域名是 `www.zzr233.online`。

**原因：** Let's Encrypt 新证书已包含 `DNS:www.zzr233.online`（SAN），但 nginx 容器里用的是旧版证书（只含 `DNS:zzr233.online`）。

**修复过程：**
1. 检查 nginx 容器证书 SANs：
   ```bash
   openssl x509 -in /home/ubuntu/nginx/ssl/zzr233.online.pem -noout -text | grep 'DNS:'
   # 只有 DNS:zzr233.online（旧的）
   # 对比 Let's Encrypt 活证书有 DNS:admin.zzr233.online, DNS:blog.zzr233.online, DNS:www.zzr233.online, DNS:zzr233.online
   ```
2. 复制新证书覆盖 nginx 容器文件：
   ```bash
   sudo cp /etc/letsencrypt/live/zzr233.online/fullchain.pem /home/ubuntu/nginx/ssl/zzr233.online.pem
   sudo cp /etc/letsencrypt/live/zzr233.online/privkey.pem /home/ubuntu/nginx/ssl/zzr233.online.key
   ```
3. 加 `www.zzr233.online` 的 HTTPS → HTTP 重定向（redirect.conf）：
   ```nginx
   server {
       listen 443 ssl http2;
       server_name www.zzr233.online;
       # 使用同样的证书（现在已包含 www）
       return 301 https://zzr233.online$request_uri;
   }
   ```
4. 默认服务器（default_server）从「直接 HTTP 返回内容」改为「全部重定向 HTTPS」（保留 `.well-known/acme-challenge/`）

**教训：**
- nginx 容器有自己的证书副本，Let's Encrypt 自动续签不更新容器内文件
- 需要定期同步或写 cron 脚本复制到 `/home/ubuntu/nginx/ssl/`
- `default_server` 不该直接返回内容，应该重定向到 HTTPS

## 2. Astro `<style is:global>` 不能穿透 React 组件 ❌

**场景：** 想隐藏移动端 `ElectricBorder` 的 canvas 动画，在 `Announcement.astro` 写 CSS：
```css
<style is:global>
@media (max-width: 767px) {
  .eb-canvas-container { display: none !important; }
}
</style>
```

**结果：** 没用。闪电还在。

**原因：** Astro 即使是 `is:global`，**仍然会给选择器加上 `[data-astro-cid-xxx]` 作用域属性**。最终的 CSS 变成：
```css
.eb-canvas-container[data-astro-cid-ovwmtxdk] { display: none !important; }
```
这个 `[data-astro-cid-ovwmtxdk]` 是 Announcement.astro 的 scope ID，`.eb-canvas-container` 实际在 ElectricBorder React 组件内部，没有这个属性，所以选择器匹配不上。

**正确方案：直接在 React 组件里判断移动端：**
```tsx
const ElectricBorder = ({ children, ... }) => {
  const [isMobile, setIsMobile] = useState(false);
  useEffect(() => { setIsMobile(window.innerWidth < 768); }, []);

  // 在动画 useEffect 里：
  useEffect(() => {
    if (isMobile) return;  // 跳过所有 canvas 创建和动画循环
    // ... canvas setup
  }, [isMobile, ...]);

  // 在 render 里：
  if (isMobile) {
    return (
      <div className="electric-border electric-border-mobile">
        <div className="eb-content">{children}</div>
      </div>
    );
  }

  // 桌面版：正常渲染 canvas + 发光层
};
```

**教训：**
- CSS 无法穿透 React 组件边界（即使 is:global）
- 要对 React 组件内部做移动端适配 → 直接改组件本身
- 用 `useState` + `window.innerWidth < 768` 判断，SSR 安全

## 3. 移动端性能优化：哪些该关 ⚡

### 3.1 ElectricBorder（闪电边框）— 组件级修复
- 桌面：canvas 动画 + 3 层发光效果
- 移动端：替换为 2px CSS 边框（`.electric-border-mobile`），性能零开销
- 加 CSS：
```css
.electric-border-mobile {
  border: 2px solid oklch(from var(--electric-border-color) l c h / 0.35);
  overflow: hidden;
}
```

### 3.2 背景视频（BackgroundPlayer）
- 移动端直接 `display: none`（CSS，有 md 断点保护）
- 视频 + 解码对移动端 GPU 负担大

### 3.3 公告 tilt 倾斜效果
- `pointermove` 事件 + `requestAnimationFrame` 持续更新 transform
- 移动端 `transform: none !important`

### 3.4 站点统计 / 站点信息
- 在 `sidebarConfig.ts` 的 `mobileBottomComponents[]` 里 `enable: false`
- 已经包含在 `mobileBottomComponents` 里的组件只在移动端显示（`md:hidden`），关掉只影响移动端

### 3.5 文章卡片 enter 按钮
- 无封面文章右侧绿色竖条按钮
- 移动端 `display: none` + 内容 `width: 100%`

| 组件 | 移动端方案 | 桌面影响 |
|------|-----------|---------|
| ElectricBorder | 跳过 canvas，CSS 边框 | ❌ 无 |
| BackgroundPlayer | `display: none` | ❌ 无 |
| 公告 tilt | `transform: none` | ❌ 无 |
| 站点统计/信息 | `enable: false`（仅 mobileBottomComponents） | ❌ 无 |
| 文章 enter 按钮 | `@media (max-width: 767px)` | ❌ 无 |

## 4. nginx 配置要点

### 4.1 配置结构
- 全部在 `/home/ubuntu/nginx/conf.d/` 下：
  - `admin.conf` — admin.zzr233.online（反代 Waline）
  - `blog.conf` — blog.zzr233.online（博客）
  - `homepage.conf` — zzr233.online（个人主页）
  - `redirect.conf` — www 重定向 + default_server HTTPS 跳转
- 挂载方式：bind mount（`:ro`）到 nginx 容器 `/etc/nginx/conf.d/`
- 修改后 `docker exec nginx-home nginx -s reload`

### 4.2 SSL 证书同步
- Let's Encrypt 活证书：`/etc/letsencrypt/live/zzr233.online/`
- nginx 容器副本：`/home/ubuntu/nginx/ssl/zzr233.online.pem`
- 续签后需手动复制：
```bash
sudo cp /etc/letsencrypt/live/zzr233.online/fullchain.pem /home/ubuntu/nginx/ssl/zzr233.online.pem
sudo cp /etc/letsencrypt/live/zzr233.online/privkey.pem /home/ubuntu/nginx/ssl/zzr233.online.key
docker exec nginx-home nginx -s reload
```

### 4.3 http2 弃用警告
```
the "listen ... http2" directive is deprecated, use the "http2" directive instead
```
不影响功能，只是新 nginx 语法的锅。可以用独立 `http2;` 指令替代。
